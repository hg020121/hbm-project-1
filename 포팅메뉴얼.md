## 3. 환경 설정

### 3-1. DB 설정 (MariaDB via Docker)

MariaDB를 Docker 컨테이너로 실행합니다.

```bash
# MariaDB 컨테이너 실행 (포트 53301 → 내부 3306 매핑)
docker run -d \
  --name hbm-mariadb \
  -e MYSQL_ROOT_PASSWORD=SqlDba-1 \
  -e MYSQL_DATABASE=hbm_integrated_db \
  -p 53301:3306 \
  mariadb:10.11
```

DB 접속 정보:

| 항목     | 값                |
| -------- | ----------------- |
| Host     | 127.0.0.1         |
| Port     | 53301             |
| User     | root              |
| Password | SqlDba-1          |
| Database | hbm_integrated_db |

> Docker Desktop이 실행 중이어야 합니다.
> 컨테이너 재시작: `docker start hbm-mariadb`

### 3-2. 백엔드 설정 (FastAPI)

**① 가상환경 생성 및 의존성 설치**

```bash
cd hbm-project

# 가상환경 생성
python -m venv venv

# 가상환경 활성화
# Windows
venv\Scripts\activate
# macOS / Linux
source venv/bin/activate

# 의존성 설치
pip install -r requirements.txt
```

**② 환경변수 설정 (`.env`)**

프로젝트 루트(`hbm-project/`)에 `.env` 파일을 생성하거나 기존 파일을 수정합니다.

```env
# DB 설정
DB_HOST=127.0.0.1
DB_PORT=53301
DB_USER=root
DB_PASS=SqlDba-1
DB_NAME=hbm_integrated_db

# OpenAI API
OPENAI_API_KEY=sk-proj-xxxx...   # 실제 API Key로 교체
CHAT_MODEL=gpt-4o
EMBED_MODEL=text-embedding-3-small

# FAISS 벡터스토어 경로 (PDF RAG)
FAISS_DIR=faiss_store

# 청크 설정
CHUNK_SIZE=900
CHUNK_OVERLAP=150
```

> `OPENAI_API_KEY`는 반드시 본인의 실제 키로 교체해야 합니다.

**③ DB 테이블 생성 및 초기 데이터 적재 (최초 1회)**

`api/` 폴더 안에 있는 SQL 파일을 **DB 툴**에서 순서대로 실행합니다.
(DBeaver, HeidiSQL, DataGrip, TablePlus 등 사용)

DB 접속 후 아래 순서로 각 파일을 열어 실행하세요:

| 순서 | 파일            | 대상 DB           | 내용                            |
| ---- | --------------- | ----------------- | ------------------------------- |
| 1    | `api/step1.sql` | hbm_fragmented_db | AS-IS 파편화 테이블 생성        |
| 2    | `api/step2.sql` | hbm_fragmented_db | AS-IS 파편화 더미 데이터 INSERT |
| 3    | `api/step3.sql` | hbm_integrated_db | **통합 DB 테이블 생성** ★ 필수  |
| 4    | `api/step4.sql` | hbm_integrated_db | 통합 DB 더미 데이터 INSERT      |

> step3, step4만 실행해도 애플리케이션 구동에는 문제없습니다.
> step1, step2는 AS-IS 파편화 현황을 재현하는 참고용 파일입니다.

### 5-2. 공정 모니터링 대시보드 (`/engineer`)

**경로:** `frontend/src/views/EngineerDashboard.vue`
**역할:** 공정연구원 — 상단 네비게이션에서 `🔧 공정 모니터링` 선택

#### KPI 카드 (4개)

| 카드        | 내용                             |
| ----------- | -------------------------------- |
| 전체 LOT    | DB에 등록된 전체 LOT 수          |
| 수율        | 완료 LOT 중 `사용` 판정 비율 (%) |
| 진행 중 LOT | 공정이 완료되지 않은 LOT 수      |
| 불량 LOT    | `사용불가` 판정 LOT 수           |

#### LOT 현황 테이블

| 기능           | 설명                                                                      |
| -------------- | ------------------------------------------------------------------------- |
| 상태 필터      | 전체 / 입고(INCOMING) / 사전분석(PRE_ANALYSIS) / 완료(DONE) 탭으로 필터링 |
| 상태 배지      | DONE(초록) / STACKING·REFLOW(노랑) / INJECTION(파랑) / 기타(빨강)         |
| 공정 단계 표시 | 적층 N회차 / 리플로우 N회차 / MUF 주입 / 완료 등 한국어 표시              |
| 상세 버튼      | LOT 상세 페이지(`/lot/:id`)로 이동                                        |
| AI 추천 버튼   | AI 추천 페이지(`/recommend/:id`)로 이동                                   |

#### 새 LOT 등록 모달

| 입력 항목    | 설명                         |
| ------------ | ---------------------------- |
| LOT ID       | 예) LOT-051                  |
| Vendor ID    | 소재 공급업체 ID, 예) HC-001 |
| 입고일       | 날짜 선택                    |
| 점도 (Pa·s)  | MUF 소재 점도값              |
| CTE (ppm/°C) | 열팽창계수                   |

> 등록 시 LOT과 입고 데이터가 함께 생성되고 상태가 `INCOMING`으로 설정됩니다.

#### 더미 데이터 생성 모달

| 입력 항목 | 설명                      |
| --------- | ------------------------- |
| LOT 개수  | 생성할 LOT 수 (1~100)     |
| 목표 수율 | 0~1 범위 (예: 0.98 = 98%) |

> Stacking 10회 + Reflow 9회 + MUF Injection + 최종결과 데이터를 자동 생성합니다.

### 5-3. LOT 상세 페이지 (`/lot/:id`)

**경로:** `frontend/src/views/LotDetail.vue`

#### 정보 패널 (2열 레이아웃)

| 패널               | 표시 정보                                              |
| ------------------ | ------------------------------------------------------ |
| 📦 입고 & 사전분석 | Vendor ID / 점도 / CTE / 입고일 / 측정 점도 / 측정 CTE |
| 📊 최종 결과       | 판정(사용/재활용/사용불가) 배지 / 최종 Void % / 분석일 |

#### Stacking 공정 이력

- **Void Trend 바 차트:** 각 stacking 회차의 void 비율을 막대 그래프로 시각화
- **이력 테이블:** seq / 압력(MPa) / Void(%) / 날짜 / 추천ID

#### VOID PATTERN 층별 히트맵

| 항목        | 설명                                                     |
| ----------- | -------------------------------------------------------- |
| 표시 조건   | stacking 이력이 1건 이상일 때 자동 표시                  |
| LOT ID 배지 | 히트맵 우상단에 현재 LOT ID 표시                         |
| 구성        | L1(하단) ~ LN(상단), 각 층 12개 셀                       |
| 색상 기준   | LOT 내 min~max 범위로 **상대 정규화** (층간 비교용)      |
| 범례        | 낮음(초록) / 중간(노랑) / 높음(빨강)                     |
| 통계        | 최소 VOID / 평균 VOID / 최대 VOID / SPEC 초과(>5%) 층 수 |
| 셀 호버     | 층 번호 + 실측 void % 툴팁 표시                          |

#### Reflow 이력 / AI 추천 이력

- Reflow: seq / 온도(°C) / 날짜
- AI 추천: 추천ID / 압력(MPa) / 온도(°C) / 생성 시간

#### 상단 버튼

| 버튼       | 기능                             |
| ---------- | -------------------------------- |
| 🤖 AI 추천 | 해당 LOT의 AI 추천 페이지로 이동 |
| ← 뒤로     | 이전 페이지로 이동               |

### 5-4. AI 추천 페이지 (`/recommend/:id`)

**경로:** `frontend/src/views/AiRecommend.vue`
**기능:** Decision Gate — 공정 시작 전 AI가 파라미터를 추천하고 진행 여부를 결정

#### 3열 레이아웃

**좌측 — LOT & 소재 요약**

| 항목                              | 내용                |
| --------------------------------- | ------------------- |
| LOT ID / Vendor / Viscosity / CTE | 입고 데이터 표시    |
| 공정 상태 배지                    | 현재 lot_status     |
| Incoming QC                       | PASS 배지 고정 표시 |

**중앙 — KPI + AI 추천**

| KPI 게이지  | 설명                                      |
| ----------- | ----------------------------------------- |
| Void Risk   | 마지막 stacking의 void % (빨강/노랑/초록) |
| 공정 진행률 | stacking 횟수 × 10%                       |
| Risk Level  | Void Risk 기반 HIGH/MEDIUM 표시           |

AI Recommendation 패널:

- `🤖 AI 추천 받기` 버튼 클릭 → GPT가 유사 LOT 분석 후 추천
- 결과: **Reflow Temp (°C)** + **Stack Pressure (MPa)**

액션 버튼:
| 버튼 | 동작 |
|------|------|
| 공정 진행 | 공정 데이터 입력 페이지(`/process/:id`)로 이동 |
| 조건 수정 후 진행 | 동일 (공정 입력으로 이동) |
| ⚠️ HOLD | HOLD 알림 표시 |

**우측 — RAG Explain Panel**

| 탭             | 내용                                     |
| -------------- | ---------------------------------------- |
| 왜 위험한가?   | AI 추천 결과의 `reason` 텍스트 표시      |
| 유사 LOT       | 현재 LOT의 Stacking void Trend 바 차트   |
| 권장 조치 근거 | 최근 3회 stacking seq / 압력 / void 표시 |

SAM 이미지 분석 (우측 하단):

- 이미지 업로드 → GPT Vision 분석 → void 추정값 + 분석 텍스트 표시

### 5-5. 공정 데이터 입력 (`/process/:id`)

**경로:** `frontend/src/views/ProcessInput.vue`

공정 단계를 선택하고 각 단계의 데이터를 수동으로 입력/등록합니다.

| 단계             | 입력 항목                                                      |
| ---------------- | -------------------------------------------------------------- |

| 🌡️ Reflow        | seq(순서) / 온도(°C) / 엔지니어 ID                             |

| 📊 최종 결과     | 최종 Void 비율 / 판정(사용/재활용/사용불가) / 엔지니어 ID      |

> 등록 성공 시 3초간 성공 메시지가 표시됩니다.

### 5-6. 품질 정밀분석 대시보드 (`/quality`)

#### KPI 카드 (4개)

| 카드      | 내용                   |
| --------- | ---------------------- |

| 컬럼      | 설명                                            |

## 8. 트러블슈팅

### 백엔드 시작 시 DB 연결 오류

```
sqlalchemy.exc.OperationalError: Can't connect to MySQL server
```

**원인:** MariaDB 컨테이너가 미실행
**해결:** `docker start hbm-mariadb` 실행 후 서버 재시작

---

### 이미지 분석 후 히트맵이 표시되지 않음

**원인:** GPT Vision 응답에 `layer_voids` 필드 누락
**해결:** `api/services/ai_service.py`의 fallback 로직으로 `void_estimate` 기반 자동 생성
스키마(`api/schemas/hbm.py`)와 라우터(`api/routers/hbm.py`)에 `layer_voids` 필드가 포함되어 있어야 합니다.

---

### `--reload` 후에도 변경사항이 반영되지 않음

**해결:** 서버 완전 종료(Ctrl+C) 후 재시작

```bash
python -m uvicorn api.main:app --host 127.0.0.1 --port 8000 --reload
```

---

### 프론트엔드 API 요청 실패 (CORS 오류)

**원인:** 백엔드 서버 미실행 또는 포트 불일치
**확인:**

1. 백엔드가 `8000` 포트에서 실행 중인지 확인
2. `vite.config.js`의 프록시 타겟이 `http://127.0.0.1:8000`인지 확인

---

### AI 추천 응답 없음 / OpenAI 오류

**원인:** `.env`의 `OPENAI_API_KEY` 미설정 또는 만료
**해결:** 유효한 OpenAI API Key로 교체 후 서버 재시작
